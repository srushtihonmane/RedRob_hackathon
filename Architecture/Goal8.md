### Goal 8 — Compute optimization — ✅ FULLY DETAILED & LOCKED (D1–D9)
**SCOPE:** the runtime/compute-engineering contract spanning the whole pipeline — defines the offline↔runtime boundary, the two execution environments, retrieval/scoring compute, determinism, the budget envelope, and end-to-end reproducibility. Owns *how* the system runs within ≤5 min / ≤16 GB / CPU / no-network (§3) while pushing all heavy work into the unbudgeted offline phase (§10.3). Does **not** own scoring logic/weights (Goals 4–6) — only their execution envelope.

**Performance-profiling findings that shaped Goal 8 (measured at true data shapes on a 2-core/3.8 GB sandbox — deliberately weaker than the 16 GB target, so all numbers are conservative):**
- **Goal 8 is not compute-bound.** Total in-budget work ≈ **1–2 s cold wall-clock vs the 300 s limit (~150–300× headroom)**; peak memory ≈ **~1.3 GB vs 16 GB (~10×)**; disk ≈ **0.5–1 GB vs 5 GB**.
- Embeddings 100k×384 f32 = **153.6 MB each** (identity+evidence = 307 MB); full `np.load` ~33 ms, mmap ~0.1 ms. Feature matrix 100k×90 = 36 MB, ~9 ms. BM25 CSC ~20M nnz ≈ 160 MB.
- Exact brute-force cosine (ideal + 5 archetypes, both embeddings) over full 100k = **~74 ms**; BM25 fixed ~40-term query via term-major CSC gather = **~0.2 ms**; top-k via `argpartition` = **~0.4 ms**; weighted aggregate 100k×90 = **~1.3 ms**.
- The single largest line item is the quantile transform: **~433 ms full-pool vs ~30 ms on the ~5–8k shortlist** → re-ranking belongs on the shortlist (D4), but even full-pool fits.
- **The shortlist is no longer a budget necessity** — even exhaustive full-pool scoring is sub-second; the cascade survives for architecture/defensibility/future-proofing, not compute (D4).

**D1 — Phase boundary contract (LOCKED): two phases; runtime is pure load-and-arithmetic.** The in-budget ranking step performs **no candidate-dependent embedding or model inference at runtime** (the ban is on *candidate-dependent* inference, not embedding in general — leaving the door open for the Sandbox path and the already-frozen JD vectors). It loads frozen artifacts → vectorized array arithmetic → union shortlist → Stage-B re-rank → Goal 5/6 gates → top-100 → Goal 7 reasoning → CSV (~1–2 s).
- **Runtime ML Policy (LOCKED, behavioral — not a package allowlist):** the ranking step is a **frozen scorer, not an ML pipeline.** It must **not instantiate, load, download, or execute any neural model or inference engine** — transformers, sentence-transformers, Torch, ONNX, GGUF, or any other — to compute candidate-dependent outputs at runtime. **ALL** neural computation (embedding generation, reranking, calibration fitting, model inference) happens **offline**; at runtime the system only loads precomputed artifacts (embeddings, indexes, feature matrices, lookup tables, calibration params) and performs deterministic retrieval/filter/score/rank/reason/CSV.
- **Dependencies are constrained by behavior, not by a fixed package list.** The only hard runtime limits are CPU-only / no-network / ≤5 min / ≤16 GB / ≤5 GB disk; **any CPU-only, offline, deterministic library is permitted** (numpy, scipy, pyarrow, pandas, joblib, …). A **lean, model-free dependency set is preferred** (smaller, more deterministic Stage-3 image) but **not mandated** — *the prior "numpy/scipy only" wording is superseded by this behavioral policy.* Decision rule (mirrors cross-cutting principle #2): **when in doubt push computation into precompute, not runtime; between two rule-compliant implementations prefer the simpler.**

**D2 — CE compute contract (LOCKED).** If enabled, CE is computed **offline over the entire released pool**; Competition Runtime performs **lookup only**; Sandbox Runtime computes CE on uploaded samples (only when enabled). **Shortlist-only precompute rejected** (no offline saving — offline is unbudgeted — and fragile coupling to the retrieval config across the 3-submission tuning). Inclusion is a **Goal 4 D9 audit decision; default OFF** until it shows measurable ranking value without increased trap exposure. **Artifact = `ce_features.npy [N, C]`** (2-D, row-aligned; C may be 1 initially; reserved cols: CE relevance score, percentile, calibrated score, margin-over-null) — extensible without an artifact-format break; active cols documented in the feature manifest.

**D3 — Retrieval = exact brute-force (LOCKED).** All full-pool similarity operations are **exact and vectorized unless profiling demonstrates a runtime violation** — exact numpy over L2-normalized embeddings (dot=cosine), top-k via `argpartition` (membership only), BM25 via term-major CSC gather, structural prefilter as a boolean mask. **ANN (FAISS/HNSW) rejected by default** — no speed need at N=100k; adds recall risk (could silently drop a ~1%-rare Tier-5 from the recall-first shortlist), nondeterminism, and a dependency that violates D1. Rejection rationale documented for Stage-5.

**D4 — Three-tier allocation; explicit cascade retained (LOCKED).** **Tier 1** (offline → runtime lookup, zero runtime compute): embeddings, BM25 index, feature matrix, frozen pool-stats, anti-profile vectors, company_table, jd_query, Goal 5 risk flags, CE features (if on). **Tier 2** (full-pool at runtime, cheap): similarity/retrieval channels to form the shortlist (~75 ms) + a cheap full-pool relevance proxy for the audit manifest. **Tier 3**: Stage-B re-ranking (criterion satisfaction, fusion, modifiers, `final_score`) on the **recall-first union shortlist only** (Goal 4 D3). The cascade is kept for **JD-v2 mirroring, future-heavy-reranker bounding, and stage ablatability — explicitly not for budget** (profiling shows full-pool also fits; that is the documented fallback). Goal 6 D2 reconciled: full `final_score` on shortlist; full-pool proxy retained for excluded-score auditing. **Fallback principle (LOCKED):** even if future revisions remove *all* shortlist-only operations (CE permanently off + fully-vectorized Stage-B), the architecture **may retain the shortlist boundary for separation-of-concerns** — absence of a computational reason is not, by itself, grounds to delete the retrieve/re-rank seam.

**D5 — Loading & memory (LOCKED).** **Full-load** densely-scanned matrices (embeddings ×2, features — we matmul every row); **mmap** sparse/selective artifacts (BM25 CSC → ~40 cols touched; the **row-addressable snippet sidecar** → ~100 rows materialized, the only meaningful memory win). Competition Runtime **never loads raw `candidates.parquet`/JSONL** (snippets + features replace it — the largest potential memory item is excluded by construction). Dtypes **f32** (embeddings/features), **int32** (ids); f16 a documented lever only if memory ever tightens. Peak ~1.0–1.5 GB vs 16 GB. **Two-phase load order (LOCKED):** (1) startup validation load — read the *small* artifacts first (manifests, frozen stats, jd_query, candidate_ids), run **hash + alignment + shape checks**; (2) only on success, load large matrices and execute. Flow `load manifests → validate → load matrices → execute` — a broken/misaligned/stale artifact fails loudly in ~100 ms instead of after hundreds of MB (consistent with Goals 1 & 3 loud-failure philosophy).

**D6 — Determinism (LOCKED).** No randomness, clocks, locale dependence, or runtime-derived statistics. Stable hashing only (`zlib.crc32`/`hashlib`, never salted `hash()`); `PYTHONHASHSEED=0`. Deterministic collection ordering (`np.unique` for unions, never set-iteration order). Frozen artifacts + frozen pool statistics. Pinned Python/NumPy/SciPy/Docker environment. Deterministic tie-break hierarchy (Goal 6 D4). **Single-thread BLAS preferred and enabled in the canonical reproduction image — a reproducibility-hardening choice, NOT the primary correctness guarantee.** Primary guarantee of identical rankings = **fixed artifacts + deterministic logic + deterministic tie-breaking**; single-thread BLAS only removes residual float-reduction jitter. **Reproduction validation:** (1) canonical Docker run → exact output-hash match; (2) semantic validation → identical ranking / top-100 membership. Binds Competition Runtime; Sandbox Runtime exempt.

**D7 — Budget accounting + guards (LOCKED).** Budget documented from measured profiling (~1–2 s cold, ~1.3 GB peak); **cold-start path is the canonical budget** (model-free import, no neural-inference engine loaded — per the D1 Runtime ML Policy). Runtime **always attempts to produce the CSV**; timing and memory ceilings are enforced in **CI/regression testing, not by self-aborting the production run** (a valid CSV <300 s still scores; a hard memory check ~8 GB stays, since exceeding 16 GB = DQ). Startup invariant checks validate artifact integrity + alignment (D5). Emit **`runtime_report.json`** every run (per-stage timings, peak RSS, bytes loaded, thread/cold flags). Budget numbers documented as **conservative** (weaker-than-target hardware). **`runtime_report.json` + manifests = the Stage-3 / Stage-5 evidence package.**

**D8 — Two runtimes, shared ranking logic, separate dependency sets (LOCKED).** **Competition Runtime** = D1 artifact-only ranking step (the submission + Stage-3 reproduction); constraints = the 5-min/16 GB/CPU/no-network contract + the D1 Runtime ML Policy (no runtime neural inference); deps = a **lean, model-free, CPU-only/offline set** (numpy/scipy/pyarrow/pandas/joblib as needed) — constrained by behavior, not a fixed allowlist. **Sandbox Runtime** = end-to-end demonstration on a ≤100 arbitrary sample (§10.5); constraints = ≤5 min CPU, no hosted-LLM calls; deps = may carry model(s) + heavier deps. They **share ranking logic (Goals 4→7 byte-identical)** but **not dependency sets**. Sandbox runs the Goal 1→3→5 builders on the sample on the fly (candidate-dependent embedding permitted here per D1), then applies **shipped frozen pool-stats** (normalization, tail-anchor calibration, concept freq, anti-profiles, company_table, jd_query, weights) — **never recomputes them from the sample** (this actively *proves* the transforms are pool-frozen). Optional pool-subset lookup fast-path. **Hosting platform NOT locked** — must satisfy §10.5 (hosted env *or* a `docker run` recipe in README; the Docker image doubles as the Stage-3 repro environment). Profiling: sandbox embed ≈ ~100×6 docs ≈ 3–12 s + optional CE ~100 pairs — well inside 5 min.

**D9 — Manifest & reproducibility (LOCKED).** Two-command repro (`precompute.py --candidates …` unbudgeted → `rank.py --artifacts … --out submission.csv` budgeted), both documented + shipped (§10.3-compliant). **`build_manifest.json` is the SOLE top-level entry point** — the one file an organizer opens first; everything hangs from it. It references every sub-manifest + report by path and carries the hash chain + provenance, e.g. `{ "candidates_sha256", "goal1_manifest", "goal2_manifest", "goal3_manifest", "goal5_manifest", "runtime_report", "ce_features_sha256?", "submission_sha256", "git_commit", "lib_versions", "model_revision", "thread_settings", "timestamps" }` — no tooling needs to discover sub-manifests independently. Startup **artifact↔pool binding assertion** (loaded artifacts' source-hash matches the candidates file + `candidate_ids` align across all artifacts; loud fail before any matrix loads — satisfies §10.3 "cover exactly the released pool"). **Reproduction tiers:** ranking-step from shipped artifacts = **bit-identical** (canonical Docker) / **semantic** (else, D6); full-from-raw (`precompute`→`rank`) = **semantic-equivalent** (embedding float nondeterminism per Goal 3 D9 documented; shipped artifacts are canonical). **Two pinned dependency sets** (Competition Runtime = lean model-free CPU-only/offline set per the D1 Runtime ML Policy — its Dockerfile = the Stage-3 image; precompute + Sandbox carry torch/transformers/sentence-transformers + CE model if enabled). Evidence package = sub-manifests + `build_manifest.json` + `runtime_report.json` + determinism self-test result.
