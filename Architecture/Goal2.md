### Goal 2 — JD understanding — ✅ FULLY DETAILED & LOCKED (D1–D8)
**SCOPE:** process the single fixed JD once, offline, heavy. Emit a **3-channel JD query** persisted as a frozen artifact: (a) curated lexical concept terms+synonyms (BM25 layer), (b) dense "ideal candidate" embedding (semantic layer), (c) structured **typed-criteria** object (rule layer). Goal 2 declares **what to look for + target semantics + which fields hold the evidence**; it does **not** assign numeric weights, thresholds, scoring formulas, or detection logic (those live in Goals 4/5). Objective is **engageable-hireability**, so behavioral signals feed relevance, not just fraud filtering.

**Data-profiling findings that shaped these decisions (all 100k profiled):**
- **Skills arrays are deliberately diluted noise** — every skill name appears ~12,000× (near-uniform). Matching `skills[].name` is worthless = the engineered keyword-stuffer trap. **Lexical channel must not lean on the skills array.**
- **Real concepts are rare and live in narrative text:** "ranking/search/rec" language in only **1.0%** of profiles, embeddings/vector **0.2%**, eval/NDCG/MRR **0.1%**; target titles (ML/AI/search/rec engineer) **0.8%**. (LLM/NLP language 51% = the "AI = recent LangChain" inflation.) → dense channel does the heavy lifting for plain-language Tier-5s.
- **`skill_assessment_scores` is the corroboration goldmine but sparse** (only 24% have any) and skews CV/speech (YOLO, CNN, OpenCV, ASR, TTS, diffusion) — that population *is* the "CV/speech primary, no NLP/IR" disqualifier; IR/PEFT/Milvus assessments are rare (~340 each) = genuine signal.
- **Geography:** 75% India, 8.5% Noida/Pune, 37.6% Tier-1 IN city → soft modifier, not a gate. **Consulting:** 9.7% career-entirely-consulting (disqualifier pop.); 61.5% have *some* consulting stint (must NOT disqualify). **YOE 5–9 = 34%, 6–8 = 18%** → soft band confirmed.
- **Engagement is the live differentiator:** last-active 32–272 days (median 137), GitHub absent 64.6%, offer-history absent 59.6%, open_to_work 35%. **Honeypots detectable:** 84 skills show expert+0-months; 25 candidates have YOE exceeding summed tenure by >5yr.
- **Takeaway:** corroboration across narrative + title + assessment + signals is everything; any single field (esp. skills) is a trap → corroboration is first-class in the schema.

**D1 — Typed-criteria schema (LOCKED): typed criteria with evidence binding (declarative *what*, not *how*).** Per-criterion fields: `id`, `criterion_type` ∈ {`hard_gate`, `strong_negative`, `must_have`, `nice_to_have`, `context_modifier`}, `polarity` (+/−), `importance` ∈ {`critical`,`high`,`medium`,`low`} (numeric weights deferred to Goal 4), `description`, `evidence_sources[]` (explicit candidate-schema field paths), `corroboration{min_sources}` (criterion-level), `match_modes[]` ⊆ {`lexical`,`semantic`,`structured`,`signal`}, `target_kind` ∈ {`presence`,`range`,`gate`,`modifier`}, optional `override_condition`, and `provenance` (JD origin, explicit or implied). *Excluded from Goal 2:* numeric weights, similarity thresholds, scoring formulas, executable predicates, detection/retrieval logic.

**D2 — Criteria catalog (LOCKED, structure):**
- **Must-haves** (positive, `presence`, kept as **5 separate** criteria for traceability; mostly lexical+semantic, `min_sources:2`): `production_embeddings_retrieval` (CRITICAL), `vector_db_hybrid_search_ops` (CRITICAL), `shipped_ranking_search_rec_at_scale` (CRITICAL; +`structured` product-not-services & company_size), `ranking_eval_frameworks` (HIGH), `strong_python` (HIGH; lean on `skill_assessment_scores`, not the diluted skills array).
- **Nice-to-haves** (positive, `presence`, LOW, `min_sources:1`): `llm_finetuning`, `learning_to_rank`, `hr_tech_marketplace`, `distributed_systems_scale`, `open_source_contributions`.
- **Disqualifiers** (negative, two-tier gates per D7): `pure_research_no_production` (hard_gate), `consulting_entire_career` (hard_gate, fires only when no product stint), `recent_llm_only_no_preml` (strong_negative + override), `senior_not_coding_18mo` (strong_negative), `title_chaser_1p5yr_hops` (strong_negative), `cv_speech_robotics_no_nlp` (strong_negative), `closedsource_5yr_no_external_validation` (strong_negative).
- **Context modifiers** (`range`/`modifier`): `experience_band` (range, 5–9 soft / 6–8 ideal), `applied_ml_years_at_product` (range, ideal 4–5), `location_noida_pune_relocate` (modifier), `notice_period` (modifier, ≤30 ideal), plus an **availability/engagement family** modeled as **atomic per-signal criteria** (`match_modes:[signal]`): open_to_work, last_active recency, recruiter_response_rate, saved_by_recruiters, interview_completion, verifications — transparent, lets Goal 6 build the engagement multiplier from explicit parts.
- **Consistency:** one umbrella `profile_internally_consistent` **hard_gate** (negative); exact honeypot detection deferred to Goal 5, but the criterion lives in the artifact so reasoning can cite it.
- **Omitted:** "between the lines" traits (writes-a-lot/async, defensible-opinions, plans-to-stay-3yr) — undetectable on templated ~460-char synthetic summaries; "plans to stay" already inverse of `title_chaser`.

**D3 — Authoring & verification (LOCKED): static, LLM-assisted, frozen, version-controlled artifact treated as source code.** Human-reviewed before commit; **no build-time or runtime LLM/network dependency** (reproduction = load the file). LLM allowed only during authoring (brainstorm/draft/surface implied criteria); final artifact human-approved. **Build-time validation:** well-formed schema, valid criterion_types/target_kinds/match_modes, unique IDs, every `evidence_sources` path exists in `candidate_schema.json`. **Per-criterion provenance** prevents criterion drift. Boundary confirmed: Goal 2 = declarative criteria; Goal 3 = candidate features, Goal 4 = scoring/fusion/weights, Goal 5 = detection rules.

**D4 — Lexical channel (LOCKED): curated discriminative concept groups, IDF-suppressed.** Concept groups (retrieval, vector-infra, ranking-eval, LLM, LTR, …) + **conservative** synonym expansion (canonical terms + well-known abbreviations/surface forms only — e.g. recommender/rec-sys, vector db/vectordb, FAISS, NDCG; **no loose paraphrases** — those are the dense channel's job). **Additive-positive only** (absence never penalizes → protects plain-language Tier-5s). Rely on **BM25 IDF** to auto-crush the diluted skill tokens. Candidate-side BM25 doc (built in Goal 3) draws from narrative text, **not** the skills array. Per-group boost magnitudes deferred to Goal 4.

**D5 — Dense channel (LOCKED): synthesized "ideal candidate" narrative → single positive vector.** Author a positive ideal-candidate narrative in the same register as a strong candidate's summary+career (positive signal only — **not** the raw JD, which is noisy/negative-polluted); embed it as **one** vector (mean of 2–3 paraphrasings allowed for robustness). **Model: `bge-small-en-v1.5` (384-d), L2-normalized, with a query-prefix convention — LOCKED for BOTH JD and candidate sides** (this resolves Goal 3's embedding-model open item; the same prefix discipline must be applied candidate-side in Goal 3).

**D6 — Anti-profile / negative anchor (LOCKED): per-archetype anti-vectors.** Author short negative narratives for disqualifier archetypes (research-only, CV/speech/robotics, consulting-only, recent-LLM-only, keyword-stuffer) and embed each → per-candidate cosine to each archetype is a feature (explainable: "resembles CV/speech archetype"). The artifact also **declares the divergence inputs** for the stuffer detector: `lexical_score − dense_cosine(ideal)`. Penalty/threshold formulas deferred to Goal 4/5.

**D7 — Gate semantics & importance (LOCKED): two-tier gates + override clauses.** `hard_gate` = absolute floor → tier 0 / excluded (internal-consistency, pure-research-no-production, consulting-entire-career); `strong_negative` = heavy, **overridable** penalty (recent-LLM-only, senior-not-coding, title-chaser, CV/speech, closed-source). Mirrors the JD's own "we will not" vs "we will probably not" wording. `override_condition` field encodes the JD's "unless" clauses declaratively (e.g. recent-LLM-only overridden by pre-LLM-ML evidence; consulting fires only with no product stint); detection still deferred to Goal 5. Importance levels set here; **importance→numeric-weight mapping owned by Goal 4.**

**D8 — Output artifact & reproducibility (LOCKED): vectors derived, text canonical.** Bundle: **`jd_query.json`** (canonical, human-readable: `schema_version`; `model{name:bge-small-en-v1.5, dim:384, normalize:L2, query_prefix}`; typed criteria with provenance; lexical concept groups+synonyms; the ideal-candidate narrative **text**; the anti-profile archetype narratives **text** + ordered labels; divergence-input declaration; references to vector files), **`jd_ideal.npy`** `[1×384]` f32 L2-normalized, **`jd_antiprofile.npy`** `[A×384]` f32 L2-normalized (row order = archetype labels), **`jd_manifest.json`** (sha256 of each file, model name+revision, dim, normalization, prefix, git commit, timestamp, schema fingerprint, build-validation status — mirrors Goal 1 manifest discipline), optional **`jd_rationale.md`** (Stage-5 defense). `.npy` vectors are regenerable from the canonical narratives + pinned model.
